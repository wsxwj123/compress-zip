# INTERFACE — compress-zip 对外接口约定

> 本文件是命令行内核的黑盒契约。测试设计只依赖本文件，不看实现代码即可照着写测试。
> 一切以「进程调用 + 参数 + 环境变量 → 退出码 + stdout/stderr」为准。
> 定稿日期 2026-07-15。改接口必须先改本文件。

---

## 0. 调用形式

内核是单个 Python 脚本 `czip.py`，两个子命令：`compress`（压缩）、`extract`（解压）。

```
python3 <项目>/czip.py compress ...
python3 <项目>/czip.py extract ...
```

- 密码一律走**环境变量 `COMPRESS_PW`**，**绝不作为命令行参数**（避免出现在 `ps` 进程列表）。
- 只在需要密码时（`--encrypt` 非 `none`，或解压加密包）才读取该变量；用完即弃，不落盘、不写日志。
- 所有中文/空格/emoji 路径按 `argv` 原样传入，脚本不做二次编解码。

## 1. compress 子命令

### 1.1 语法

```
czip.py compress --format {zip,7z,targz} --encrypt {none,aes,zipcrypto}
                 [--out ARCHIVE_PATH] [--] PATH [PATH ...]
```

| 参数 | 必填 | 说明 |
|---|---|---|
| `PATH ...` | 是（≥1） | 要打包的文件/文件夹，可多个混选。以 `-` 开头的路径前置 `--`。 |
| `--format` | 是 | `zip` / `7z` / `targz`（tar.gz）。 |
| `--encrypt` | 是 | `none` 不加密 / `aes` AES-256 / `zipcrypto` 传统兼容加密。 |
| `--out` | 否 | 输出压缩包的**完整路径**。省略时按 1.3 规则自动推导。 |
| 环境变量 `COMPRESS_PW` | 见下 | `--encrypt` 非 `none` 时**必须非空**；`none` 时忽略。 |

### 1.2 格式 × 加密 合法组合

| 格式 \ 加密 | none | aes | zipcrypto |
|---|---|---|---|
| zip | ✅ | ✅ | ✅ |
| 7z | ✅ | ✅ | ❌ 退出码 5 |
| targz | ✅ | ❌ 退出码 5 | ❌ 退出码 5 |

**ZipCrypto 单文件大小上限**：手搓 ZipCrypto 写入器**不实现 zip64**，单个成员文件 **> 4 GiB（0xFFFFFFFF 字节）时退 5**（提示改用 AES）。原因：zip64 只为这一条冷路径实现不划算；需要大文件加密的走 AES（pyzipper 自动 zip64）。`none`/`aes` 路径由 zipfile/pyzipper 自动处理 zip64，无此限制。

### 1.3 输出路径推导（省略 `--out` 时）

- **输出目录 = 第一个 `PATH` 所在的父目录**（始终如此，与其它输入在不在同一目录无关——多选跨目录时"源目录"不唯一，故统一锚定第一个输入，结果确定可测）。
- 基础名：
  - **单个输入** → 该输入的名字（去掉末尾斜杠）；
  - **多个输入** → 固定基础名 `归档`（不再用父目录名——"只选了部分内容却以父目录命名"会误导用户，且多选跨目录时父目录本就不唯一）。
- 扩展名 = zip→`.zip`，7z→`.7z`，targz→`.tar.gz`（多选 zip 即 `归档.zip`）。

### 1.3.2 包内结构（不加顶层前缀 + 不写 macOS 元数据）

- **不加顶层文件夹前缀**：每个所选项以其自身名字作为包内顶层条目（arcname = 所选项名字），对齐 macOS Finder 的"压缩"行为。
  - 例：选中文件夹 `foo` → 包内顶层就是 `foo/…`，**不套** `foo/foo/…`；选中 `a.txt`、`b.txt` 两个文件 → 包内顶层是 `a.txt`、`b.txt` 两项。
  - **注意**：这改变了旧假设。旧版靠顶层前缀让"原地解压回嵌套目录"来避免覆盖源；现在不加前缀，**解压端的不覆盖改由 Zip Slip 校验 + 智能布局 + 顶层同名改名共同保证**（见 §2.3），压缩端不再承担这个职责。
- **不写 macOS 元数据**：`.DS_Store` 与 `__MACOSX/` 一律不写进包。否则既污染包，又会让解压端"顶层项计数"误判（多出隐藏顶层项 → 本该铺开的包被误判成多项而套文件夹）。

### 1.3.1 symlink 处理策略（声明）

输入含符号链接时**跟随链接、存目标文件内容**（不存链接本身），三格式一致。理由：避免存链接本身带来的跨平台失效与指向包外的安全问题。

### 1.4 不删不覆盖（硬约束）

- 目标压缩包路径（无论来自 `--out` 还是自动推导）若**已存在**，自动改名避让：
  `name.zip` → `name-1.zip` → `name-2.zip` …（在扩展名前插 `-N`；`.tar.gz` 视为整体扩展名，即 `name-1.tar.gz`）。
- 全程只读输入、只写新压缩包，**绝不删除或覆盖任何已有文件**。

### 1.5 成功输出

- 退出码 `0`。
- **stdout 打印一行**：实际生成的压缩包**绝对路径**（改名避让后的最终路径），无其它多余文字。
- stderr 为空。

### 1.6 中文名保证（头号验收，可自动验证）

对 `--format zip` 且含中文名的输入，生成的 zip 包内每个中文条目：
- 文件名以 **UTF-8 字节**存储；
- **通用位标志 bit 11（0x800）置位**（`ZipInfo.flag_bits & 0x800 == 0x800`）。

此保证对 `none` / `aes` / `zipcrypto` 三种加密方式**都成立**（含手搓 ZipCrypto 路径）。

## 2. extract 子命令

### 2.1 语法

```
czip.py extract [--layout {auto,flatten,folder}] [--dest DIR] [--] ARCHIVE
```

| 参数 | 必填 | 说明 |
|---|---|---|
| `ARCHIVE` | 是 | 要解压的压缩包路径。 |
| `--layout` | 否 | 落地布局，默认 `auto`。`auto`=智能默认（单顶层项铺开、多顶层项套包名文件夹）；`flatten`=强制把包内顶层项直接铺进目标目录；`folder`=强制套一层以包名命名的文件夹。三者含义见 §2.3。 |
| `--dest` | 否 | 解压目标**父目录**。省略时 = `ARCHIVE` 所在父目录（"解到压缩包旁边"）。 |
| 环境变量 `COMPRESS_PW` | 见下 | 仅当包被加密时使用；非空则用它解密，为空则按未加密尝试。 |

### 2.2 支持格式（按扩展名/内容识别）

`.zip`（普通/AES/ZipCrypto 加密均可读）、`.7z`（含加密）、`.tar.gz` / `.tgz`、`.rar`（需系统 `unar`）。

**带密码 rar 不支持**：rarfile 靠 `unar` 子进程解压，带密码时会把密码作为命令行参数传给 unar（落进 `ps`），违背"密码绝不进 argv/ps"承诺，且无廉价的非 argv 传密码路径。故解压前用 `rarfile.RarFile.needs_password()` 探测，加密 rar 直接退 5（见 §4）。普通 rar 正常解。

### 2.3 解压落地布局与不覆盖（硬约束）

**"顶层项"定义（钉死，`auto` 判据）**：把每个成员路径先做反斜杠归一化与 `.DS_Store`/`__MACOSX/` 过滤，再取其**第一路径段**，去重后的集合即"顶层项集合"，**其基数=顶层项数**。
- 例：成员 `foo/a.txt`、`foo/b.txt`、`foo/`（无论有无显式目录条目）→ 第一路径段都是 `foo` → 去重后 **1 项**（铺开）。成员 `a.txt`、`b.txt` → `{a.txt, b.txt}` → **2 项**（套壳）。
- **不能按 `namelist()` 条数数**（那样"单文件夹装 N 文件"会被当成 N 项误套壳，还击穿自压包原地解压的不覆盖论证）。

**默认解压位置**：省略 `--dest` 时 = `ARCHIVE` 所在父目录（解到压缩包旁边）；给了 `--dest` 则解到该目录下。

**落地布局（由 `--layout` 决定，默认 `auto`）**：
- `auto`（智能默认，对齐 The Unarchiver / Keka）：
  - **0 个顶层项**（空包，或过滤 macOS 元数据后剩 0）→ 不写任何文件，**退 1** + stderr `压缩包为空`（见 §4.2）。
  - **恰好 1 个顶层项** → **铺开**：该顶层项直接落在 `<dest>/` 下（不套壳，避免 `归档/归档` 套娃）。
  - **≥2 个顶层项** → **套文件夹**：新建 `<dest>/<ARCHIVE 去扩展名的名字>/`，全部内容解进去（避免散文件污染当前目录）。
- `flatten`（强制铺开）：所有顶层项直接落在 `<dest>/` 下，不套壳；0 项同样退 1。
- `folder`（强制套文件夹）：无条件新建 `<dest>/<ARCHIVE 去扩展名的名字>/`，全部内容解进去；0 项同样退 1。

**不覆盖 = 顶层项级前缀重写（硬约束）**：
- 改名作用在**顶层项整体**上，**做法是前缀重写、不逐成员判重**：先按落地目录确定每个顶层项的最终名（撞名则 `foo` → `foo-1` → `foo-2` …，文件在扩展名前插 `-N`、目录直接追加 `-N`），**再把该顶层项下所有成员的落地路径前缀统一从 `foo/**` 重写为 `foo-1/**` 后才落盘**。
  - 铺开时（`auto` 单项 / `flatten`）：逐个顶层项判 `<dest>/<名字>` 是否已存在，撞名则整项前缀重写为 `-N`。
  - 套文件夹时（`auto` 多项 / `folder`）：只判壳文件夹 `<dest>/<包名>/` 是否存在，撞名则壳整体重写 `包名` → `包名-N`；壳内不再逐项判。
- **禁止退化成"逐成员判 `<dest>/foo/x` 是否存在"**——那样会把包内 `foo/` 逐个文件 merge 进已存在的源 `foo/`、覆盖源里的同名文件。必须整项定名后统一重写前缀。
- **自动改名、不弹窗**。全程只读压缩包、只写新落地项，**绝不删除或覆盖任何已有文件**。
- **原地解压自压包的自洽**：本工具压缩不加顶层前缀，故 `foo.zip`（顶层单项 `foo`，内含多个文件）在源目录原地 `auto` 解压时，落地目标 `<dir>/foo` 撞上源 `foo` → 整项前缀重写落 `foo-1/**`，源 `foo/` 及其内文件零改动。这正是"无顶层前缀"后不覆盖的保证方式（替代旧版靠嵌套避让）。

**Zip Slip + 软/硬链接目录穿越防护（安全红线，覆盖 zip / 7z / targz / rar 全格式）**：
- **拒绝一切软链接 / 硬链接成员**：任何成员是 symlink 或 hardlink（tar 的 `SYMTYPE`/`LNKTYPE`、zip 的 Unix 模式位 `S_ISLNK`、7z/rar 的链接属性）→ **整包拒绝、退 1**。理由：软链接成员可先建一条指向 `<dest>` 外的链（如 `link` → `/Users/x/.ssh`），后续成员 `link/authorized_keys` 就被 OS 顺链写到目录外——纯字符串路径校验拦不住。本工具压缩侧本就"跟随链接存目标内容、不存链接本身"（§1.3.1），自压包无链接成员，此拒绝只影响外部包，代价可接受。
- **路径校验，逐成员在真实落点复核（不止解压前预扫）**：对每个成员，反斜杠归一化后要求**路径不含 `..` 段、不以 `/` 开头（非绝对路径）**；并在**即将写入前**对其真实落点求 `os.path.realpath`，确认仍以落地目录为前缀（把"解压前对字符串预扫一次"改为"每写一个成员对真实落点复核一次"，配合上一条拒链接，堵住"链接先落地、后续成员顺链逃逸"）。任一成员逃逸 → **整包拒绝、退 1**，落地目录外零文件写入。
- 校验对所有格式一视同仁：zip/pyzipper 虽内建净化 `..`，仍再统一过一遍；7z、tar.gz、rar 成员逐个校验（本机 Python 3.10.11 < 3.10.12，`tarfile` 的 `filter='data'` 不可用，故手动校验 + 手动拒链接）。
- **反斜杠归一化**：外部包成员路径含 `\`（Windows 风格）的，先把 `\` 转 `/` 再解析层级与建目录，避免解出名字里带反斜杠的怪文件；转换后同样过上面全部校验。

### 2.4 成功输出

- 退出码 `0`。
- **stdout 打印一行绝对路径**：套文件夹布局 → 壳文件夹路径（`<dest>/<包名>` 改名避让后的最终路径）；铺开布局 → 落地目录 `<dest>` 的路径。
- stderr 为空。

## 3. 退出码定义（compress / extract 通用）

每个码的含义（正文定义，不只列在表里）：

| 码 | 含义 | 输出约定 |
|---|---|---|
| **0** | 成功 | stdout 打一行绝对路径，stderr 空 |
| **1** | 内部/意外错误：磁盘 IO 失败、压缩包损坏/不完整、Zip Slip 逃逸成员被拒、含链接成员被拒、空包、未预期异常 | stderr 打一行 `错误: <说明>`，stdout 空 |
| **2** | 用法错误：缺参数、非法 `--format`/`--encrypt` 取值 | argparse 自带消息（含 `usage:`）进 stderr，stdout 空 |
| **3** | 输入不存在：某个 `PATH` / `ARCHIVE` 磁盘上找不到 | 同 1 的输出形态 |
| **4** | 密码错误：需要密码但 `COMPRESS_PW` 为空；或解压时密码错误 | 同上 |
| **5** | 不支持的组合/操作：见 §4 明细 | 同上 |
| **6** | 缺后端依赖：`unar` 不在 PATH、必需 pip 库导入失败 | 同上 |

**校验顺序（决定多错并发时返回哪个码，测试据此断言）**：
参数合法性（→2/5）→ 密码存在性（→4）→ 输入存在性（→3）→ 实际压/解（运行期错误 →1/4/5/6）。

## 4. 完整错误契约（子命令 × 每类非法输入）

- 业务错误统一：**stdout 为空**，**stderr 打印一行** `错误: <说明>`，退对应码。
- argparse 用法错误（码 2）：消息由 argparse 决定（含 `usage:` 前缀），进 stderr。
- 测试用**子串**断言 stderr（下表括号/引号内为稳定子串），避免因标点/路径拼接细节脆断。

### 4.1 compress

| 非法输入 | 退出码 | stderr 稳定子串 |
|---|---|---|
| 缺 `--format`/`--encrypt`、非法枚举值、无 `PATH` | 2 | argparse `usage:` |
| targz 选了加密（aes/zipcrypto） | 5 | `tar.gz 不支持加密` |
| 7z 选了 zipcrypto | 5 | `7z 不支持 ZipCrypto 加密` |
| zipcrypto 且单文件 > 4 GiB | 5 | `ZipCrypto 不支持超过 4GB 的单文件，请改用 AES` |
| `--encrypt` 非 none 但 `COMPRESS_PW` 为空 | 4 | `该加密方式需要密码，但未提供` |
| 某个 `PATH` 不存在 | 3 | `找不到输入: <path>` |
| 输出目录不可写 / 磁盘写失败 | 1 | `写入失败` |
| 缺 pip 依赖库（pyzipper/py7zr） | 6 | `缺少依赖库 <name>，请先 pip install <name>` |

### 4.2 extract

| 非法输入 | 退出码 | stderr 稳定子串 |
|---|---|---|
| 缺 `ARCHIVE`、`--layout` 非法枚举值 | 2 | argparse `usage:` |
| `ARCHIVE` 不存在 | 3 | `找不到输入: <path>` |
| 不支持的解压格式（未知扩展/内容） | 5 | `不支持的压缩包格式` |
| **带密码 rar**（needs_password 为真） | 5 | `不支持带密码的 rar 解压` |
| 解 rar 但 `unar` 不在 PATH | 6 | `解压 rar 需要 unar，请先运行 brew install unar` |
| 缺 pip 依赖库（rarfile/pyzipper/py7zr） | 6 | `缺少依赖库 <name>，请先 pip install <name>` |
| 加密包但 `COMPRESS_PW` 为空，或密码错误 | 4 | `密码错误或压缩包已损坏` |
| 包损坏 / 不完整 / 截断 | 1 | `压缩包损坏或不完整` |
| 成员越界（Zip Slip：含 `../`、绝对路径、或反斜杠归一化后仍逃出落地目录），全格式适用 | 1 | `压缩包含非法路径，已拒绝` |
| **含软链接/硬链接成员**（防顺链逃逸），全格式适用 | 1 | `压缩包含链接成员，已拒绝` |
| **空包**：无成员，或过滤 `.DS_Store`/`__MACOSX` 后剩 0 个顶层项 | 1 | `压缩包为空` |
| 目标目录不可写 / 磁盘写失败 | 1 | `写入失败` |

## 5. 可自动化验收要点（给测试设计的锚点）

1. **中文不乱码（头号）**：`compress --format zip` 含中文名 → 用标准库 `zipfile` 打开生成包，断言每个中文条目 `flag_bits & 0x800 == 0x800` 且 `info.filename` 为正确中文（说明以 UTF-8 存储）。三种加密方式各测一遍。
2. **加解密闭环**：aes / zipcrypto 各压一个包 → 用正确 `COMPRESS_PW` 解出、内容一致（退出 0）；用错误密码解 → 退出 4 且 stderr 含 `密码错误`。
3. **不覆盖/顶层改名**：同目标名压两次 → 第二次输出 `-1` 后缀路径，原文件仍在；`auto` 解压两次 → 第二次落地顶层项前缀重写 `-1`（铺开改单项名、套壳改壳文件夹名），原有内容不被覆盖。**原地解自压包必须用"单文件夹装多文件"的包**：断言源 `foo/` 及其内每个文件零改动、新内容落 `foo-1/**`（专防退化成逐成员 merge 覆盖源）。
4. **多文件混选 + 命名**：单选 → 包名=该项名字；**多选（无论同目录/跨目录）→ 包名=`归档`**（§1.3）。输出目录=第一个输入的父目录。解出目录树完整。
5. **包内结构**：自压包内**无顶层文件夹前缀**（选 `foo` 文件夹 → 包顶层为 `foo/…` 而非 `foo/foo/…`）；包内**不含** `.DS_Store` / `__MACOSX/`。
6. **解压布局（§2.3）**：**单文件夹装多文件的包（顶层项=1）→ auto 铺开**（关键：证明按"第一路径段去重"数项、而非 namelist 条数）；多顶层项包 → 套 `<dest>/<包名>/`；**空包 / 仅含 .DS_Store 的包 → 退 1 `压缩包为空`**；`--layout flatten` 强制铺开、`folder` 强制套壳，各验落地形态。
7. **边界**：名字带空格/emoji、**空文件夹**（能压能解、解出后空目录仍在）、symlink（跟随存目标内容）、大文件；非法 `--format` / `--layout` → 退出 2。
8. **格式×加密非法组合**：targz+aes → 5；7z+zipcrypto → 5；zipcrypto+>4GiB 单文件 → 5（构造/桩测，不必真造 4G）。
9. **安全边界（全格式）**：① 含 `../`、绝对路径、或反斜杠越界成员的包（zip/7z/tar/rar 各测）→ 退 1 `压缩包含非法路径`、dest 外零落地；② **含软链接/硬链接成员的包 → 退 1 `压缩包含链接成员`**（重点造"软链指向 dest 外 + 后续成员顺链写"的经典逃逸包，断言 dest 外零落地）；③ 反斜杠成员在合法范围内则归一化为 `/` 正常解出；④ 带密码 rar → 退 5。
8. **健壮性**：截断/损坏包解压 → 退 1，不崩栈。
